You are implementing **Milestone 3.1 (Imported Past Papers)** of QForge. Goal in one line:
**let real historical exams influence repetition control by recording them as `papers` rows
(`origin=imported`) the generator avoids — subject-wide.**

## Objective

Close the M3 gap where "exclude last N papers" only saw QForge-generated papers. Add an
admin-recorded **imported past exam** that lands in the rolling last-N window for every teacher of
that subject, so a freshly generated paper avoids questions that were on real exams.

## Scope / deliverables

**Backend (Laravel)** — this milestone **deliberately re-opens the schema** (own migration).
- *Schema:* a migration making `papers.blueprint_id` **nullable** and adding
  `papers.origin` enum `('generated','imported')` default `'generated'`. No change to
  `paper_questions`. Add `origin` to the `Paper` model `$fillable`.
- *Engine:* change **only** `PaperGenerator::lastNExclusion()` so the rolling window is
  `where('subject_id', …)->where(fn($q) => $q->where('owner_id', $blueprint->owner_id)->orWhere('origin','imported'))`,
  still ordered by `generated_at` desc then `id` desc, still `limit(lastNPapers)`. Keep it **pure
  rolling** (imported papers age out normally — no always-exclude). Nothing else in the algorithm
  changes.
- *Service:* `App\Services\ImportedPaperService::record(Subject $subject, string $name,
  CarbonInterface $examDate, array $existingQuestionIds, array $newQuestions, User $uploader): Paper`.
  In one transaction: create any `newQuestions` (`source=extracted`, `status=approved`, tagged to
  their `unit_id`); collect all linked ids; create the `papers` row (`origin=imported`,
  `blueprint_id=null`, `owner_id=$uploader->id`, `generated_at=$examDate`, `status=saved`,
  `total_marks` = Σ linked marks, `duration=0`); create `paper_questions` (`section_label="Imported"`,
  sequential `display_no`, marks/unit from each question, `is_ai=false`); `increment('used_count')`
  for the linked questions.
- *API:* `POST /subjects/{subject}/past-papers` — **admin-only**, body
  `{ name, exam_date (required), question_ids?[], new_questions?[{text,type,marks,unit_id}] }` →
  `ImportedPaperService::record(...)` → `201 { paper }`.

**Out of scope (do not build):** no frontend wiring (imported papers stay out of teacher
History/analytics — both remain owner-scoped on generated papers); no PDF parsing (that is M4 — the
upload job will call this same service later); no "always-exclude / pinned" imported papers.

## Constraints & conventions

- Laravel owns this end to end — no Python (CLAUDE.md).
- Imported papers must **not** appear in `GET /papers` history or `GET /papers/analytics` (keep those
  owner-scoped on generated papers as they are today).
- Run all commands via `sudo docker compose exec qforge_app …` (see `docs/CONVENTIONS.md`); start the
  stack with `echo 'rohitshakya' | sudo -S docker compose up -d` if needed.
- **Stop and ask before**: changing any table other than `papers`, touching the generation algorithm
  beyond `lastNExclusion`, or adding a dependency. Only make the changes described here — do not
  refactor or add abstractions beyond this scope.

## Acceptance / verification

- `sudo docker compose exec qforge_app php artisan test` — all existing tests stay green, plus:
  - a generated paper **excludes** questions from an `imported` past exam, **cross-teacher**
    (an admin-recorded exam suppresses for teacher A *and* teacher B);
  - imported papers are **absent** from `GET /papers` history and `GET /papers/analytics` counts;
  - `record(...)` with mixed existing ids + inline `new_questions` creates the paper, links all,
    bumps `used_count`, and sets `generated_at = exam_date`;
  - rolling: an imported exam ages out once `lastNPapers` newer papers exist.
- `sudo docker compose exec qforge_app php artisan migrate:fresh --seed` runs clean.
